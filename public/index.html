This is a known Decap CMS issue with how it handles the body field. The CMS expects the markdown body to be stored as a field called body in the frontmatter, but Hugo content files store the body as plain content below the frontmatter — not as a frontmatter field.

So the file looks like this: markdown

title: “Kontakt”

actual content here ← CMS can’t see this

But Decap expects: markdown

title: “Kontakt” body: “actual content here” ← CMS looks here

The fix is adding body_field or rather setting the correct field. Actually the proper Decap fix is adding this to each file collection entry in config.yml: yaml

  • label: “Kontakt” name: “kontakt” file: “content/pages/kontakt.md” fields:
    • { label: “Title”, name: “title”, widget: “string” }
    • { label: “Body”, name: “body”, widget: “markdown” }
